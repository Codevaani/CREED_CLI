🚀 AI Coding CLI Architecture & Plan

A comprehensive plan for building a natural language coding CLI using TypeScript and Bun.

1. Core Architecture & Tech Stack

  • Runtime & Package Manager: Bun (for ultra-fast startup times and native TS support).
  • Language: TypeScript (strict typing for robustness).
  • CLI Framework: commander or clack for elegant terminal UI.
  • LLM Integration: Official SDKs (e.g., @google/genai, OpenAI SDK) for tool calling and reasoning.
  • File System: Bun's native Bun.file() and standard Node fs/promises.

2. Sub-Agent System Design

To keep the main context window lean and handle complex tasks, the CLI will delegate work to specialized sub-agents:

  • Orchestrator (Main Agent): Handles user input, maintains session history, and routes tasks to tools or sub-agents.
  • Investigator Agent: Specializes in navigating the codebase, reading files, and summarizing architecture (using `glob`, `grep`, `read_file`).
  • Coder Agent: Focuses purely on writing, replacing, and refactoring code based on specific instructions.
  • Runner/Terminal Agent: Executes shell commands (tests, builds, lints) and analyzes the output for errors.

3. Development Phases

Phase 1: CLI Setup & Basic Loop

  • Initialize Bun + TS project.
  • Set up REPL (Read-Eval-Print Loop) using standard input/output.
  • Integrate basic LLM API call with a placeholder system prompt.

Phase 2: Tooling Infrastructure

  • Parse the provided tool.json to register available functions with the LLM.
  • Implement core tools locally: read_file, write_file, run_shell_command.
  • Implement tool execution dispatcher to handle LLM tool calls and return results.

Phase 3: Multi-Agent Orchestration

  • Implement Sub-Agent routing logic.
  • Define specific system prompts for Investigator, Coder, and Runner agents.
  • Create communication bridge so Sub-Agents can return summaries back to the Orchestrator.

Phase 4: Advanced Features & Polish

  • Implement streaming responses for better UX.
  • Add context management (truncating old history to save tokens).
  • Integrate custom system prompts.
  • Add terminal UI elements (spinners, colors, markdown rendering).
